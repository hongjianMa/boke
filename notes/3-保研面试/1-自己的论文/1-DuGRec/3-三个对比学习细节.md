# 三个对比学习的细节

三个对比学习都使用同一个核心 **InfoNCE** 形式：
$$
\mathcal L_{\mathrm{NCE}}(A,B)
=
-\frac{1}{B}\sum_{i=1}^{B}
\log
\frac{\exp(\operatorname{sim}(A_i,B_i)/\tau)}
{\sum_{j=1}^{B}\exp(\operatorname{sim}(A_i,B_j)/\tau)}
$$
通俗理解：

- 分子：同一个实体的两个相关表示，即正样本。
- 分母：该表示与当前 batch 中所有候选表示的相似度。
- $j\neq i$ 的表示作为负样本。
- $\tau$ 是温度系数，控制模型区分正负样本的敏感程度。

代码中的统一实现是：

```python
def _nce(self, A, B, tau):
    logits = (A @ B.t()) / tau
    targets = torch.arange(logits.size(0), device=logits.device)
    return F.cross_entropy(logits, targets)
```

因为 $A_i$ 和 $B_i$ 在相同位置，所以相似度矩阵的**对角线是正样本，非对角线是批内负样本**。

> `sim()` 表示 **相似度函数 similarity function**，用来衡量两个向量有多相似。
>
> 在你的论文和代码中，`sim()` 实际上主要使用的是**余弦相似度**：
> $$
> \operatorname{sim}(a,b)
> =
> \frac{a^\top b}{\|a\|_2\|b\|_2}
> $$
> 其中：
>
> - $a^\top b$：两个向量的点积。
> - $\|a\|_2$、$\|b\|_2$：两个向量的长度。
> - 结果越大，说明两个向量方向越接近。
>
> 余弦相似度通常位于：
> $$
> [-1,1]
> $$
>
> - 接近 $1$：方向相同，非常相似。
> - 接近 $0$：方向近似垂直，相关性较弱。
> - 接近 $-1$：方向相反。

## 1. 用户偏好对比学习 $L_{clu}$

### 目的

让同一个用户在视觉、文本和融合语义视角下的偏好表示保持一致，从而稳定用户的个性化模态注意力。

### 公式

$$
\mathcal L_{clu}
=
\sum_{(p,q)\in\{(v,t),(t,s)\}}
-\log
\frac{
\exp(\langle z_u^p,z_u^q\rangle/\tau)
}{
\sum_{u'\in\mathcal B_u}
\exp(\langle z_u^p,z_{u'}^q\rangle/\tau)
}
$$

其中：

- $z_u^v$：用户 $u$ 的视觉偏好表示。
- $z_u^t$：用户 $u$ 的文本偏好表示。
- $z_u^s$：用户 $u$ 的融合语义表示。

### 正负样本

以视觉—文本对比为例：

- 正样本：$(z_u^v,z_u^t)$，同一个用户的视觉和文本表示。
- 负样本：$(z_u^v,z_{u'}^t)$，当前 batch 中其他用户的文本表示。

### 代码实现

首先在 `_cl_fuse_users()` 中投影并归一化：

```python
z_v = F.normalize(self.u_proj_v(v_rep), dim=1)
z_t = F.normalize(self.u_proj_t(t_rep), dim=1)
z_s = F.normalize(self.u_proj_s(s_rep), dim=1)

self._zu_v, self._zu_t, self._zu_s = z_v, z_t, z_s
```

然后 `_user_cl_total()` 选取当前 batch 用户并计算：

```python
zv = self._zu_v[users]
zt = self._zu_t[users]
zs = self._zu_s[users]

cl = self._nce(zv, zt, self.loss.clu_tau)
cl += self._nce(zt, zs, self.loss.clu_tau)
```

当前默认对比模态对是：

```python
pairs = {'vt', 'ts'}
```

即视觉—文本和文本—融合；视觉—融合 `vs` 在代码中支持，但默认未启用。

## 2. 物品模态对比学习 $L_{cli}$

### 目的

让同一个物品的图像、文本和融合表示语义一致，减小不同模态之间的语义偏差。

### 公式

$$
\mathcal L_{cli}
=
\sum_{(p,q)\in\{(v,t),(t,s)\}}
-\log
\frac{
\exp(\langle z_i^p,z_i^q\rangle/\tau)
}{
\sum_{j\in\mathcal B_i}
\exp(\langle z_i^p,z_j^q\rangle/\tau)
}
$$

### 正负样本

以视觉—文本对比为例：

- 正样本：$(z_i^v,z_i^t)$，同一个物品的视觉和文本表示。
- 负样本：$(z_i^v,z_j^t)$，batch 中其他物品的文本表示。

它与用户对比学习结构相同，区别只是对比对象从“用户”变成了“物品”。

### 代码实现

在 `forward()` 中，将物品表示切分为三个模态：

```python
v_i = item_rep[:, :d]
t_i = item_rep[:, d:2 * d]
s_i = item_rep[:, 2 * d:]
```

然后分别投影并归一化：

```python
self._zi_iv = F.normalize(self.i_proj_v(v_i), dim=1)
self._zi_it = F.normalize(self.i_proj_t(t_i), dim=1)
self._zi_is = F.normalize(self.i_proj_s(s_i), dim=1)
```

训练时先取当前 batch 的正负物品并去重：

```python
batch_items = torch.unique(
    torch.cat([pos_items, neg_items], dim=0)
)
```

再在 `_item_mod_cl_total()` 中计算：

```python
zv = self._zi_iv[batch_items]
zt = self._zi_it[batch_items]
zs = self._zi_is[batch_items]

cl = self._nce(zv, zt, self.loss.cli_tau)
cl += self._nce(zt, zs, self.loss.cli_tau)
```

默认也是使用 `vt` 和 `ts` 两组跨模态对比。论文中的物品模态对比目标，就是拉近同一物品的跨模态表示，推远不同物品的表示。

## 3. 图扰动对比学习 $L_{clg}$

### 目的

让同一个用户或物品在受到随机扰动后，表示仍然保持稳定，提高模型对噪声和缺失信息的鲁棒性。

### 公式

用户侧：
$$
\mathcal L_{clg}^{u}
=
-\frac{1}{|\mathcal B_u|}
\sum_{u\in\mathcal B_u}
\log
\frac{
\exp(\langle u^1,u^2\rangle/\tau)
}{
\sum_{u'\in\mathcal B_u}
\exp(\langle u^1,{u'}^2\rangle/\tau)
}
$$
物品侧：
$$
\mathcal L_{clg}^{i}
=
-\frac{1}{|\mathcal B_i|}
\sum_{i\in\mathcal B_i}
\log
\frac{
\exp(\langle i^1,i^2\rangle/\tau)
}{
\sum_{j\in\mathcal B_i}
\exp(\langle i^1,j^2\rangle/\tau)
}
$$
总损失：
$$
\mathcal L_{clg}
=
\mathcal L_{clg}^{u}
+
\mathcal L_{clg}^{i}
$$

### 正负样本

用户侧：

- 正样本：同一个用户的两个随机扰动视图 $(u^1,u^2)$。
- 负样本：该用户的第一个视图与其他用户的第二个视图 $(u^1,{u'}^2)$。

物品侧：

- 正样本：同一个物品的两个随机扰动视图 $(i^1,i^2)$。
- 负样本：该物品与 batch 中其他物品的扰动表示 $(i^1,j^2)$。

### 代码实现

代码默认采用特征维度随机 Mask：

```python
p = self.gpert_mask_p

m1 = (torch.rand_like(x) > p).float()
m2 = (torch.rand_like(x) > p).float()

e1 = F.normalize(x * m1 / (1.0 - p), dim=1)
e2 = F.normalize(x * m2 / (1.0 - p), dim=1)
```

即对同一表示进行两次独立随机掩码，生成两个视图。

随后分别处理用户和物品：

```python
u = ua_embeddings[users]
i = ia_embeddings[batch_items]

u1, u2 = _mk_views(u)
i1, i2 = _mk_views(i)

ssl_u = self._nce(u1, u2, self.gpert_tau)
ssl_i = self._nce(i1, i2, self.gpert_tau)
```

最后加权：

```
reg = self.gpert_lambda * (ssl_u + ssl_i)
```

代码也支持加入高斯噪声：

```python
e1 = F.normalize(x + sigma * torch.randn_like(x), dim=1)
e2 = F.normalize(x + sigma * torch.randn_like(x), dim=1)
```

但当前配置固定为：

```
self.gpert_mode = 'mask'
```

所以实际运行的是随机掩码增强。论文将其描述为同一用户或物品在两个随机扰动视图之间保持一致，并将其他批内实体作为负样本。 

## 三者的核心区别

| 对比学习  | 对比对象           | 正样本                       | 负样本                   | 解决问题               |
| --------- | ------------------ | ---------------------------- | ------------------------ | ---------------------- |
| $L_{clu}$ | 同一用户的不同模态 | 同一用户的跨模态表示         | 其他用户的模态表示       | 用户偏好跨模态不一致   |
| $L_{cli}$ | 同一物品的不同模态 | 同一物品的跨模态表示         | 其他物品的模态表示       | 物品图文语义偏差       |
| $L_{clg}$ | 同一实体的不同扰动 | 同一用户或物品的两个扰动视图 | 其他用户或物品的扰动视图 | 表示对噪声和扰动不稳定 |
