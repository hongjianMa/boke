# 【论文阅读】MDVT: Enhancing Multimodal Recommendation with Model-Agnostic Multimodal-Driven Virtual Triplets

MDVT：通过与模型无关的多模态驱动的虚拟三元组增强多模态推荐





## 摘要

现在方法对多模态信息利用不够，把他当作侧信息。

在本文中，我们提出了 MDVT，一种与模型无关的方法，它**构造多模态驱动的虚拟三元组以提供有价值的监督信号**，有效缓解多模态推荐系统中的数据稀疏问题。为了确保高质量的虚拟三元组，我们引入了三种量身定制的预热阈值策略：**静态、动态和混合。**

静态预热阈值策略详尽地搜索最佳预热时期数，但非常耗时且计算量大。

动态预热阈值策略根据损耗趋势调整预热时间，提高效率，但可能会失去最佳性能。

混合策略结合了两者，使用动态策略来找到近似最佳的预热周期数，然后在狭窄的超参数空间中使用静态策略对其进行细化。

一旦满足预热阈值，虚拟三元组将通过我们增强的成对损失函数用于联合模型优化，而不会导致明显的梯度偏差。



## 引言

现在方法利用多模态内容来增强表征，但是利用不够，他们主要专注于使用多模式内容改进项目表示，而**用户表示仍然仅从历史交互中学习。**

为了利用这种洞察力，我们引入了多模态驱动的虚拟三元组（MDVT），这是一种与模型无关的新颖方法，可基于多模态信息构建虚拟三元组。这些虚拟三元组提供信息丰富的监督信号，有效缓解多模态推荐系统中的数据稀疏问题。

一个关键的挑战是，与项目不同，用户在推荐场景中没有固有的多模态信息。用户的模态表示必须通过随机初始化嵌入并通过模型优化对其进行细化来从头开始学习。因此，用户和项目模态表示之间的初始相似性可能无法提供高质量的监督信号。

为了解决这个问题，我们引入了三种量身定制的预热阈值策略：

- Static Warm-up Threshold Strategy： 该策略详尽地搜索最佳的预热周期数，确保在构建虚拟三元组之前充分学习用户模态表示。虽然有效，但由于需要彻底的超参数调整，因此非常耗时且计算密集。
- Dynamic Warm-up Threshold Strategy: 该策略根据训练过程中损失变化的趋势来调整预热期。它通过减少大量超参数调整的需要来提高效率，自动确定何时充分学习用户表示。然而，与静态策略相比，它可能并不总能找到最佳的预热周期数。
- Hybrid Warm-up Threshold Strategy:  混合策略结合了两种策略的优点，首先采用动态策略来找到近似最佳的预热周期数，然后在狭窄的超参数空间内应用静态策略。这可以实现高效的训练，并在计算成本和性能优化之间取得平衡。

### Multimodal-Driven Virtual Triplets

为了更好地插入不同的多模式推荐模型，我们根据最终的聚合表示构建虚拟三元组。

我们将不同多模态推荐模型的最终表示聚合过程简化为：
$$
\bar{\mathbf e}_u = F(\bar{\mathbf e}_{u m}\mid m\in\mathcal M),\qquad
\bar{\mathbf e}_i = F(\bar{\mathbf e}_{i m}\mid m\in\mathcal M), \tag{4}
$$
其中 $F(\cdot)$ 表示模态表示融合（fusion）操作。这里，融合后的表示包含有价值的多模态信息 $[41,44,45]$，可用于构建**模型无关**的“多模态驱动虚拟三元组”。为了计算每个用户节点的 top-$n$ 正/负表示以构造虚拟三元组，我们使用余弦相似度维护一个用户—物品表示相似度矩阵，形式化为：
$$
S_{u,i}=\frac{\bar{\mathbf e}_u^{\top}\bar{\mathbf e}_i}{\lVert \bar{\mathbf e}_u\rVert\,\lVert \bar{\mathbf e}_i\rVert}. \tag{5}
$$
基于该相似度矩阵，对于每个用户，我们选择最相似的 $n$ 个物品和最不相似的 $n$ 个物品来构造虚拟三元组：
$$
\mathcal D^V_{u,i^+}=\mathrm{Max}_n\!\left(S_{u,i^*}\mid i^*\in\mathcal I\right),\qquad
\mathcal D^V_{u,i^-}=\mathrm{Max}_n\!\left(-S_{u,i^*}\mid i^*\in\mathcal I\right), \tag{6}
$$
其中 $\mathrm{Max}_n(\cdot)$ 表示 top-$n$ 相似度筛选操作。对用户 $u$ 而言，$\mathcal D^V_{u,i^+}$ 和 $\mathcal D^V_{u,i^-}$ 分别包含 $n$ 个正样本物品和 $n$ 个负样本物品。因此，我们构造一个新的虚拟三元组训练数据集 $\mathcal D^V$，并随模型优化过程动态更新。对于该虚拟三元组训练数据集 $\mathcal D^V$ 中、以用户 $u$ 为中心的每个三元组，可表示为：
$$
\mathcal D^V_u = \left(u,\ \mathcal D^V_{u,i^+},\ \mathcal D^V_{u,i^-}\right).
$$

### Threshold Strategies

值得注意的是，与项目天然拥有图像和文本描述等多模态信息不同，推荐场景中的**用户并不天生具有多模态信息。**

因此，用户表示最初是随机的，并在训练过程中逐步细化，该模型需要足够的预热时期来确保表示得到充分优化，以构建高质量的三元组。

### 1) Static Threshold（静态阈值）

- **做法**：提前设定一个固定 epoch $T_S$。训练到第 $T_S$ 轮后，才开启虚拟三元组。
- **优点**：最稳、最好控。
- **缺点**：要靠经验/网格搜索找 $T_S$，成本高，不同数据/模型要重调。

### 2) Dynamic Threshold（动态阈值）

- **做法**：不设定固定 epoch，而是看训练过程是否“趋于稳定”。当 **loss 的下降幅度**（变化率）小于某个阈值 $g$ 时，就认为模型表示已经靠谱，立刻开启虚拟三元组。
- **优点**：省去大量调参，自动化。
- **缺点**：loss 有噪声时可能误判（太早开会引入错误虚拟样本，太晚开又浪费收益）。

### 3) Hybrid Threshold（混合阈值）

- **做法**：先用 Dynamic 找到一个大概的开启点 $T_{\text{cur}}$，然后只在它附近 $[T_{\text{cur}}-s,\; T_{\text{cur}}+s]$ 做小范围 Static 搜索，挑表现最好的阈值。
- **优点**：比纯 Static 便宜很多，但通常接近甚至达到最优。
- **缺点**：仍然需要一点点搜索（不过范围很小）。



## Enhanced Pair-wise Loss Function

**Enhanced Pair-wise Loss** 就是在原来的 BPR 两两排序损失上，**再加一条“虚拟三元组”的两两排序监督**，并用权重把两部分尺度对齐。

- **原始 BPR（真实交互三元组）**：让用户 $u$ 对真实正样本 $i^+$ 的打分高于负样本 $i^-$

$$
L_{\text{bpr}}=-\log\sigma\!\big(\hat y_{u,i^+}-\hat y_{u,i^-}\big)
$$

- **虚拟 BPR（MDVT 生成的虚拟正/虚拟负集合）**：用相似度矩阵挑出来的 top-$n$ 虚拟正集合 $\mathcal D^V_{u,i^+}$ 和 bottom-$n$ 虚拟负集合 $\mathcal D^V_{u,i^-}$（通常先做均值聚合成一个“虚拟正/负表示”），同样要求虚拟正打分更高：

$$
L_{\text{vbpr}}=-\log\sigma\!\big(\hat y_{u,\bar i^+}-\hat y_{u,\bar i^-}\big)
$$

- **增强后的总损失（关键点：尺度对齐）**：

$$
L=(1-\lambda)L_{\text{bpr}}+\lambda L_{\text{vbpr}}
$$

这里 $\lambda$ 控制虚拟监督的强度，避免虚拟损失一加进来就主导梯度、导致训练不稳。

一句话：**它用“真实交互排序 + 虚拟相似度排序”共同训练，让稀疏交互下的用户表示更准、更稳**
